# Hướng dẫn đọc và phát triển Frontend

Frontend được viết bằng React và Vite. Phần lớn màn hình quản lý dùng chung một bộ component bảng để tránh lặp lại logic thêm, sửa, xóa, tìm kiếm và phân trang.

## Luồng dữ liệu chính

```text
View của từng chức năng
        │ truyền apiModule + columns
        ▼
DataTablePage
        ├── useEntityTable: tải dữ liệu, tìm kiếm và CRUD
        ├── DataSearchCard: ô tìm kiếm
        ├── DataTableCard: hiển thị bảng và các nút thao tác
        ├── EntityFormDialog: form thêm/sửa
        └── TablePagination: chuyển trang
                │
                ▼
             lib/api.js
                │
                ▼
          Backend REST API
```

Mục đích của cách tách này là để mỗi file chỉ có một trách nhiệm:

- `views/`: khai báo một màn hình cần những cột nào và hiển thị dữ liệu ra sao.
- `hooks/useEntityTable.js`: giữ state và xử lý tải/thêm/sửa/xóa/tìm kiếm dữ liệu.
- `components/common/DataTable/`: chứa giao diện bảng, form, chi tiết, xác nhận xóa và phân trang.
- `lib/api.js`: là nơi duy nhất làm việc trực tiếp với Axios và endpoint backend.
- `utils/`: các hàm thuần dùng lại ở nhiều nơi, ví dụ format ngày, validation và tạo lookup.

## Cách đọc một màn hình quản lý

Ví dụ `src/views/theloai/TheLoaiView.jsx`:

1. `apiModule="theloai"` cho biết dữ liệu được lấy từ `/api/theloai`.
2. `columns` mô tả khóa dữ liệu, nhãn hiển thị, loại input và khóa chính.
3. `DataTablePage` tự lo tải dữ liệu và các thao tác CRUD dựa trên cấu hình đó.

Một cấu hình cột cơ bản:

```jsx
{
  key: "MaTL",
  label: "Mã thể loại",
  primaryKey: true,
  autoGenerated: true,
}
```

Các thuộc tính thường dùng:

| Thuộc tính | Ý nghĩa |
| --- | --- |
| `key` | Tên trường đúng với dữ liệu backend |
| `label` / `displayLabel` | Nhãn hiển thị trong form hoặc chi tiết |
| `tableLabel` | Nhãn ngắn dùng riêng ở bảng |
| `primaryKey` | Trường khóa chính; hỗ trợ cả khóa ghép |
| `autoGenerated` | Tự gợi ý mã tiếp theo và khóa ô nhập |
| `inputType` | Loại input như `text`, `number`, `date` |
| `options` | Biến trường thành danh sách lựa chọn |
| `displayValue` | Hàm đổi dữ liệu thô thành nội dung dễ đọc |
| `formHidden` / `tableHidden` | Ẩn trường khỏi form hoặc bảng |
| `badge` | Hiển thị giá trị dạng nhãn trạng thái |

## Thêm một màn hình quản lý mới

Tạo view mới và chỉ khai báo cấu hình:

```jsx
import DataTablePage from "@/components/common/DataTablePage";

function ViDuView() {
  return (
    <DataTablePage
      apiModule="vidu"
      columns={[
        {
          key: "MaViDu",
          label: "Mã ví dụ",
          primaryKey: true,
          autoGenerated: true,
        },
        {
          key: "TenViDu",
          label: "Tên ví dụ",
        },
      ]}
      entityName="Ví dụ"
      title="Quản lý Ví dụ"
    />
  );
}

export default ViDuView;
```

Sau đó thêm route vào `src/routes/AppRoutes.jsx` và mục điều hướng vào sidebar nếu cần.

Nếu màn hình cần danh sách từ module khác để làm ô chọn, dùng `useApiLists`. Ví dụ màn hình sách tải tác giả, thể loại và nhà xuất bản rồi chuyển chúng thành `options`.

## Quy ước để code dễ hiểu

- View chỉ nên chứa cấu hình và logic riêng của nghiệp vụ đó.
- Logic gọi API dùng chung đặt trong hook hoặc `lib/api.js`, không viết lại trong từng view.
- Hàm biến đổi dữ liệu không phụ thuộc React nên đặt trong `utils/`.
- Đặt tên theo nghiệp vụ đầy đủ; tránh tên một chữ hoặc viết tắt khó đoán.
- Khi một component bắt đầu xử lý nhiều nhiệm vụ, tách phần state/logic thành hook và phần giao diện nhỏ thành component.
